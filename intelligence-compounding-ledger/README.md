# Intelligence Compounding Ledger™

A lightweight operating ledger for turning AI conversations, ideas, decisions, and observations into compounding intelligence assets.

## Overview

The **Intelligence Compounding Ledger™** is a simple tracking system for operators who use AI as more than a chatbot.

Instead of letting useful insights disappear inside scattered conversations, the ledger forces each meaningful signal into a structured path:

```text
Signal → Pattern → Decision → Action → Asset → Proof → Upgrade
```

The goal is to make intelligence compound over time.

Every important AI interaction should become one of four things:

1. **Action**
2. **Asset**
3. **Evidence**
4. **System Upgrade**

If it becomes none of those, it should be archived or discarded.

## Core Thesis

AI does not replace intelligence.

AI reveals whether intelligence has a system.

The value is not one good answer. The value is the loop that captures insight, tests it, routes it, and improves the next decision.

## Features

- Capture important AI, business, research, and life signals.
- Identify repeated patterns across conversations and decisions.
- Convert insights into actions, assets, proof, or system upgrades.
- Track whether an idea has been captured, tested, built, published, monetized, archived, or discarded.
- Prevent AI output clutter by assigning every output a route.
- Support operating-system style workflows for creators, founders, strategists, researchers, and AI operators.

## Core Ledger Fields

| Field | Purpose |
|---|---|
| **Date** | When the signal was captured. |
| **Signal** | What showed up or became visible. |
| **Pattern** | Whether this connects to something repeated. |
| **Decision** | What this changes or clarifies. |
| **Action** | What should be done next. |
| **Asset** | What permanent artifact this could become. |
| **Proof** | What evidence supports its value. |
| **Upgrade** | What workflow, system, rule, or process improves because of it. |
| **Route** | Where the output should go next. |
| **Status** | Current stage of the item. |
| **Next Move** | The next concrete step. |

## Route Codes

| Route | Use When |
|---|---|
| **Execute** | The signal requires immediate action. |
| **Evergreen** | The signal should become a reusable asset. |
| **Share** | The signal should become public-facing content. |
| **Audit** | The signal needs verification, review, or risk testing. |
| **Offer** | The signal can become part of a product, service, diagnostic, sprint, or sales asset. |
| **Archive** | The signal is useful but not currently active. |
| **Discard** | The signal is weak, duplicate, noisy, unsupported, or not useful. |
| **Restrict** | The signal is private, sensitive, protected, or not for public use. |

## Status Codes

| Status | Meaning |
|---|---|
| **Captured** | The signal has been noticed but not processed. |
| **Patterned** | The signal connects to a recurring theme. |
| **Decided** | The signal changed a decision or priority. |
| **Built** | The signal became a usable asset, workflow, tool, or system component. |
| **Published** | The signal became public or shareable content. |
| **Monetized** | The signal became part of an offer, product, sprint, or service. |
| **Archived** | The signal is preserved but inactive. |
| **Discarded** | The signal is not worth keeping. |

## Recommended Workflow

Use this ledger after any meaningful AI session.

### 1. Capture the Signal

Ask:

```text
What did this conversation reveal?
```

### 2. Identify the Pattern

Ask:

```text
Have I seen this before?
Is this a one-off idea or a repeated signal?
```

### 3. Make the Decision

Ask:

```text
What does this change?
Does this affect a priority, offer, workflow, belief, or next action?
```

### 4. Assign a Route

Choose one:

```text
Execute / Evergreen / Share / Audit / Offer / Archive / Discard / Restrict
```

### 5. Set the Next Move

Ask:

```text
What is the next physical or digital action?
```

## Minimum Daily Use

At the end of the day, log one row.

```text
Signal:
Pattern:
Decision:
Action:
Asset:
Proof:
Upgrade:
Route:
Status:

```

This keeps the system lightweight and prevents overbuilding.
